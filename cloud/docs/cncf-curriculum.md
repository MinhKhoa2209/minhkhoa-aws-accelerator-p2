# CNCF Curriculum

## Purpose

This note is not meant for immediate W8 execution. It helps you decide where to go next after W8 in the broader cloud-native learning path.

## What To Remember

- W8 is only an introduction to the cloud-native space.
- `CKA` and `CKAD` are natural next directions for platform and DevOps-oriented learning.
- `KCNA` is a lighter entry point if you want broader cloud-native context first.
- The curriculum should be used to map knowledge gaps, not followed blindly as a checklist.

## How It Applies to W8

- After `cloud/w8/day-2`, `day-3`, and `lab`, come back to the curriculum and identify which areas you already touched.
- If networking, workloads, or debugging still feel weak, that is a signal to strengthen practice before chasing certification objectives.

## Practical Guidance

- use the curriculum to choose your next learning direction
- do not rush into certification tracks if your hands-on foundation is still weak
- get the lab stable first, then map your work to curriculum objectives

## Self-Check

- After W8, are you leaning more toward `CKA`, `CKAD`, or `KCNA`?
- Which Kubernetes area still feels weakest to you?
- Can you already debug an application in a cluster, or can you only apply manifests?
